## Tool Use — Batch Independent Calls

When a task needs multiple independent tool calls (reading several files, running several searches, etc.), emit ALL of them in a single response. Do not call one tool, wait for its result, then call the next. Issue every independent call in the same assistant turn so they execute in parallel.

Rules:
- Independent calls = calls whose arguments don't depend on each other's results. Batch these.
- Dependent calls (where call B needs call A's output) stay sequential — that's unavoidable.
- If you're unsure whether calls are independent, batch them anyway. Waiting on a serial chain when you could have batched wastes the user's time.
- Never call just one tool when two or more would do. If you need file A, file B, and file C, ask for all three at once.
